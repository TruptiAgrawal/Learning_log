# Amazon Clone Static Page - Learning Log

**Project Type:** Frontend Web Development  
**Technologies:** HTML5, CSS3, JavaScript (ES6)
**Deployed Link:** https://sh0pie.netlify.app/

## Project Overview
Built a responsive Amazon homepage clone with interactive features including search, shopping cart, wishlist, and category filtering - all using vanilla JavaScript without frameworks.

## Key Learnings

### 1. DOM Manipulation & Event Handling
- Implemented real-time search filtering using event listeners
- Dynamically rendered product cards from JavaScript objects
- Managed multiple interactive features (cart, wishlist, filters) simultaneously

### 2. LocalStorage API
- Persisted shopping cart data across browser sessions
- Stored wishlist items for user preference retention
- Learned data serialization with JSON.stringify() and JSON.parse()

### 3. Responsive Design
- Created mobile-first hamburger menu with slide-out navigation
- Used CSS Flexbox for flexible layouts
- Implemented media queries for tablet and mobile breakpoints
- Ensured touch-friendly UI elements for mobile devices

### 4. State Management (Vanilla JS)
- Managed application state without frameworks
- Synchronized UI with data changes
- Handled cart count updates and price calculations

### 5. CSS Techniques
- Hover effects and smooth transitions
- Overlay backgrounds for modals
- Icon integration with Font Awesome
- Consistent styling across components

### 6. User Experience Features
- Real-time search with case-insensitive filtering
- Visual feedback for user actions (heart icon toggle, active categories)
- Cart modal with add/remove functionality
- Category-based product filtering

## Challenges Overcome
- Managing multiple interactive features without state management libraries
- Ensuring data persistence with localStorage
- Creating responsive navigation that works across devices
- Synchronizing cart count and total price updates

## Skills Developed
- Vanilla JavaScript proficiency
- CSS layout techniques (Flexbox)
- Browser storage APIs
- Responsive web design
- Event-driven programming

## Future Improvements
- Add product detail modal
- Implement quantity selector in cart
- Add price range filter and sort functionality
- Integrate with backend/AWS services
- Add user authentication

## Takeaways
This project reinforced the importance of understanding core web technologies before jumping to frameworks. Building features like cart management and state synchronization from scratch provided deep insights into how modern frameworks solve these problems.
